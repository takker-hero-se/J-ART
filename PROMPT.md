# Kotonoha Guard — 元プロンプト（保存版）

> このファイルは、本プロジェクト `Kotonoha Guard` を生成するために使用した元の指示を記録したものです。

作成したWebアプリの名前は **J-ART (Japanese Adversarial Red-Team framework)**（当初案: Kotonoha Guard → Sakura Jailbreak Matrix から変更）

---

あなたは優秀なシニアセキュリティエンジニアおよびMLOpsアーキテクトです。
GitHub Actions とオープンソースのLLM脆弱性スキャナー（garak または deepeval/deepteam）の思想を組み合わせ、「日本語環境におけるRAGシステムおよびガードレールのMITRE ATLASフレームワーク準拠の脆弱性耐性評価リーダーボード（静的Webサイト）」を完全自動で構築したいと考えています。

既存の大規模ベンダーのリーダーボードと明確に差別化するため、以下の【必須要件】を満たす最小限かつ拡張性の高い実証コード（PoC）とCI/CDパイプラインを生成してください。

### 【必須要件】
1. **検証対象（ターゲット）の構造化:**
   - 単体のLLM API（OpenAI, Anthropic, Gemini等）だけでなく、システムプロンプトや簡易的なRAG（検索コンテキストの注入）、またはガードレールを挟んだ「アプリケーション構造層」を疑似的に再現してテストできる設計にすること。

2. **日本語特有の攻撃（難読化・エッジケース）の自動シミュレーション:**
   - 英語ベースのデフォルトテストではなく、MITRE ATLASの「間接的プロンプトインジェクション(AML.T0061)」や「制御奪取」を日本語環境で検証するため、ベースとなる攻撃プロンプトを動的に変換するラッパー関数を実装してください。
   - 例：慇懃無礼なビジネス日本語への変換、縦書き・改行挟み、ギャル文字/難読化、二枚舌プロンプト（前提条件で騙す）など、日本独自の3〜5パターンの変形ロジックを含めること。

3. **コストパフォーマンス（コスパ）指標の自動計測:**
   - 単に防御の成否を判定するだけでなく、テスト実行時に消費した「推論トークン数（入力/出力）」または「想定APIコスト」を算出・記録するロジックを実装してください。
   - 「セキュリティスコア（防御成功率）÷ 100万トークンあたりのコスト」のような、実務で最も重要となる「コスト効率よく安全を担保できているか」を可視化する独自指標（コスパスコア）を計算に含めること。

4. **自動化パイプライン (GitHub Actions):**
   - 定期実行（cron）および手動実行（workflow_dispatch）に対応した `.github/workflows/eval.yml` を作成すること。
   - テスト実行、結果のJSON保存、最新データに基づいた `index.html`（静的Webページ）の自動生成、GitHub Pages への自動デプロイまでをワンストップで行うこと。
   - 各種APIキーなどの機密情報は GitHub Secrets (`OPENAI_API_KEY` 等) から安全に読み込むこと。

5. **出力成果物 (静的Web画面):**
   - Pythonスクリプトによって自動生成される、シンプルでモダンな `index.html`。
   - 各モデル・システム構成ごとの「ATLAS防御成功率」と「コスパスコア」を並び替え可能なランキング形式のテーブルで表示し、どのような日本語攻撃で突破されたかのログ（Reasoning）も確認できる簡潔なUI（Tailwind CSS等のCDNを利用可）。

---

### 【出力してほしいもの】
1. **`requirements.txt`**: 必要なライブラリ一式（deepeval, pyyaml, openai 等、実装に最適なものを選択）。
2. **`run_assessment.py`**: 日本語変形ロジックとコスト計測ロジックを内包し、各モデル/構成に対してテストを実行して結果を `results.json` にラップして出力するメインスクリプト。
3. **`generate_site.py`**: `results.json` を読み込み、コスパ指標やランキング表、攻撃ログを含む `index.html` を自動生成するスクリプト。
4. **`.github/workflows/eval.yml`**: 上記を全て自動実行して GitHub Pages にパブリッシュするワークフロー定義。

コードは省略せず、そのままコピー＆ペーストで動作する本番クオリティのコード（プレースホルダーではなく、具体的な日本語変形ロジックや計算式が実装された状態）で出力してください。

   - 列項目：「順位」「モデル / システム構成」「ATLAS防御成功率（%）」「100万トークンあたりのコスト（USD）」「コスパスコア（独自指標）」「ステータス（SAFE / WARNING / VULN などのカラーバッジ）」。

3. **インタラクティブな詳細ログ（アコーディオンUI）:**

4. **フッター:**
   - GitHubリポジトリへのリンクや、コミュニティからの新たな日本語攻撃プレイブック（テストケース）のプルリクエストを歓迎する旨を記載。

上記のデザイン要件を完全に満たし、プレースホルダー（省略）なしでそのまま既存のコードと差し替えられる「generate_site.py」の完全なソースコードを再出力してください。

## 追記: run_assessment.py マルチプロバイダ対応プロンプト（2026-06-10）

J-ARTのリーダーボードとして実用的なベンチマークにするため、以下のモデル群を切り替え、またはループで回せるように共通のエンドポイント呼び出し関数（ラッパー）を実装してください。

### 【対応させるAIサービス・モデル構成】

### 【実装要件】

- **エラーハンドリングとレートリミット対策:** APIのタイムアウトやレートリミット（429）に備え、簡易的なリトライロジック（`time.sleep`）を挟むなど、GitHub Actions上で途中で止まらないタフなコードにしてください。
